Getting Started

Installation

When using Compass, one of the first things you will need to do is create your main HTML file, which will hold the compass-body element. This element is where Compass will render your desired view. 
<html>
	<head>
		<title>Compass.js Demo</title>
	</head>
	<body>
		<compass-body></compass-body>
	</body>
</html>
To install compass, include it like you would with any other javascript file.
<script type="text/javascript" src="js/compass.js"></script>

Basic Usage

To actually start using compass, you will need to call the compass.map() function. compass.map() takes one parameter, an object specifying where each route should lead. Each route can either directly link to an HTML file, or it can link to a fuction. 
<script>
	function loadUserPage() {
		// Code to load user page will go here.
	}

	compass.map({
		"/" : "views/index.html",
		"/about" : "views/about.html",
		"/user" : loadUserPage
	});
</script>
Compass uses hash based routing to resolve the issue of browsers making requests to non-existent pages. For you, this means that you when linking to your newly created routes, you will need to add # to your links, like this: 
<a href="/">Home</a>
<a href="#/about">About</a>
<a href="#/user">User Dashboard</a>
So that's the basics of using compass. Now go build something neat, or stick around and learn about some more useful features.

Additional Features

URL Parameters

URL parameters are segments of the url which can be used to hold specific values passed which we may need to properly direct the user to the correct resource. For example, if we receive a request to load the page #/posts/45, we know that we want to load a page to display post #45. Rather than define a route for every post up to 45, we can instead define one route like this: #/posts/:postID, and decide what to with the postID value later. 
<script>
	function loadPost(params) {
		// Params is an object containing our URL parameters.
		var postID = params.postID;
		console.log("Loading post:", postID);
		compass.render("posts/" + postID + ".html");
	}

	compass.map({
		"/posts" : "views/posts.html",
		"#/post/:postID" : loadPost
	});
</script>
URL parameters open up a lot of possibilities regarding what you can do in terms of page routing. You can also have multiple parameters defined in one URL.

Page Rendering

On their own, URL parameters aren't very helpful, since you need to actually show some content to your users. The compass.render() function allows you to utilize Compass' native rendering function to replace compass-body with your content. 
<script>
	function loadPost(params) {
		var postID = params.postID;
		compass.render("posts/" + postID + ".html");
	}
</script>

Always Functions

If for some reason you need to have a piece of code run everytime you load a page, you have a couple of options. You can define either a compass.beforeLoad() function or a compass.onLoad() function. These functions work as you expect, being called before and after rendering the page, and are useful if you have any constant UI elements or a need to validate user identity on every page. 
<script>
	compass.onLoad = function() {
		// Called after the content is rendered.
		window.scrollTo(0,0);
	}

	compass.beforeLoad = function() {
		// Called before the content is rendered.
	}
</script>

404 Page

Sometimes your user won't always end up where you want them to be. If this is the case you will probably want to show them some sort of error page. This can be accomplished by defining a 404 route in your compass.map call. 
<script>
	compass.map({
		"/" : pageFunctions.index,
		"/docs": pageFunctions.docs,
		"404" : "views/404error.html"
	});
</script>
